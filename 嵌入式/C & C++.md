# 缓冲`IO`
缓冲 `I/O `是指数据在写入到最终目的地（屏幕、文件、网络）之前，**先暂存到内存中的缓冲区**。当缓冲区填满、程序结束或遇到特定条件时，才真正执行 `I/O `操作。

## 缓冲的作用
### 直接`IO`, 无缓冲
```c
printf("A") → 系统调用 → 写入屏幕（耗时 ~1μs）
printf("B") → 系统调用 → 写入屏幕（耗时 ~1μs）
```
- 每次调用都触发系统调用，开销大
### 缓冲`IO`
```c
printf("A") → 写入缓冲区（内存操作，极快）
printf("B") → 写入缓冲区
```
- 缓冲区满 → 一次系统调用写入所有数据（耗时 ~1μs，但只调用一次）
- **效率提升**：将多次系统调用合并为一次，性能提升可达 **10~100 倍**

## 三种缓冲类型
| 缓冲类型 | 触发条件                        | 适用流                | 特点              |
| ---- | --------------------------- | ------------------ | --------------- |
| 全缓冲  | 缓冲区满或 `fflush()`/`fclose()` | 文件流（`fopen` 打开的文件） | 效率最高，延迟最大       |
| 行缓冲  | 遇到换行符 `\n` 或缓冲区满            | 终端输出流（`stdout`）    | 保证每行及时显示        |
| 无缓冲  | 立即输出，不缓存                    | 标准错误流（`stderr`）    | 实时输出，保证错误信息及时显示 |
- 全缓冲需要`fflush()`或者`fclose()`才会进行真正的`IO`
- 行缓冲可以用`\n`或`fflush()`来立刻进行`IO`

## 强制刷新的办法
|方法|说明|适用场景|
|---|---|---|
|`fflush(stream)`|显式刷新指定流|需要立即看到输出时|
|`fflush(NULL)`|刷新所有输出流|程序关键点|
|`fclose(stream)`|关闭流（隐含刷新）|文件操作结束|
|`exit()` / `return 0`|程序结束（隐含刷新所有缓冲区）|正常结束|
|`setbuf()` / `setvbuf()`|改变缓冲类型|自定义缓冲行为|
