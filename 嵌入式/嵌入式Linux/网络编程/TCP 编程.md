# `listen`函数
```c
#include <sys/socket.h>
int listen(int sockfd, int backlog);
```
**返回值**
- 成功：返回 **0**
- 失败：返回 **-1**，并设置 `errno` 错误码

## 参数详解
- `sockfd`: 套接字文件描述符, 由`socket()`创建
- `backlog`: 连接队列长度
指定**最大等待连接数**，即内核允许的未处理连接请求的最大数量
```
┌─────────────────────────────────────────────────────────────┐
│                        连接队列                              |
├─────────────────────────────────────────────────────────────┤
│  ┌─────┐ ┌─────┐ ┌─────┐ ┌─────┐ ┌─────┐ ┌─────┐            │
│  │  1  │ │  2  │ │  3  │ │ ... │ │  N  │ │  排队中...        │
│  └─────┘ └─────┘ └─────┘ └─────┘ └─────┘ └─────┘            │
│                                                             │
│  如果队列已满，新的连接请求会被拒绝或忽略                          │
└─────────────────────────────────────────────────────────────┘
```


# `connect`函数
`connect()` 是 `TCP` 客户端用于**主动向服务器发起连接请求**的函数。它是客户端网络编程的起点，负责完成 `TCP` 三次握手的发起过程。
```c
#include <sys/socket.h>
int connect(int sockfd, const struct sockaddr *addr, socklen_t addrlen);
```
- 返回值: 成功返回 0, 失败返回 -1

**`addr` —  服务器地址结构体**
指向 `struct sockaddr_in`（`IPv4`）或 `struct sockaddr_in6`（`IPv6`）的指针，用于指定目标服务器的 IP 地址和端口号。



# `accept`函数
`accept()` 是 `TCP` 服务器端用于**从已完成连接队列中取出一个连接请求**，并创建一个新的套接字用于与该客户端通信的函数。它是服务器端握手过程的最后一步
```c
#include <sys/socket.h>
int accept(int sockfd, struct sockaddr *addr, socklen_t *addrlen);
```
**返回值**：
- 成功：返回一个**新的套接字文件描述符**（≥0），用于与客户端通信。
- 失败：返回 **-1**，并设置 `errno` 错误码。

 **`addr` —  客户端地址结构（输出参数**）
用于存放客户端的 IP 地址和端口号。
- 如果不需要客户端信息，可以传入 `NULL`。
- 实际类型为 `struct sockaddr_in`（`IPv4`），需强制转换。


# `send`函数
`send()` 是 `TCP` 套接字中最常用的数据传输函数。它们用于在已建立连接的套接字上发送数据。
```c
#include <sys/socket.h>
ssize_t send(int sockfd, const void *buf, size_t len, int flags);
```
**返回值**：
- 成功：返回实际发送的字节数（可能小于 `len`）。
- 失败：返回 **-1**，并设置 `errno` 错误码。

## 参数

| 参数       | 说明                                     |
| -------- | -------------------------------------- |
| `sockfd` | 已连接的套接字（由 `accept()` 或 `connect()` 返回） |
| `buf`    | 指向要发送数据的缓冲区                            |
| `len`    | 要发送的数据长度（字节数）                          |
| `flags`  | 标志位，通常设为 `0`                           |
## `flags` 常用值

|标志|说明|
|---|---|
|`0`|默认行为，阻塞直到数据发送完成|
|`MSG_DONTWAIT`|非阻塞模式，数据无法立即发送时返回 `EAGAIN`|
|`MSG_NOSIGNAL`|连接断开时不产生 `SIGPIPE` 信号（防止进程意外终止）|
|`MSG_OOB`|发送带外数据（紧急数据）|


# `recv`函数
`recv()` 是 `TCP` 套接字中最常用的数据传输函数。它们用于在已建立连接的套接字上接收数据。
```c
#include <sys/socket.h>
ssize_t recv(int sockfd, void *buf, size_t len, int flags);
```
**返回值**：
- 成功：返回实际接收的字节数（可能小于 `len`）。
- 返回 `0`：对方已关闭连接（正常关闭）。
- 失败：返回 **-1**，并设置 `errno` 错误码。

## 参数详解

|参数|说明|
|---|---|
|`sockfd`|已连接的套接字|
|`buf`|存放接收数据的缓冲区|
|`len`|缓冲区大小（最多能接收的字节数）|
|`flags`|标志位，通常设为 `0`|

## `flags` 常用值

|标志|说明|
|---|---|
|`0`|默认行为，阻塞直到有数据到达|
|`MSG_DONTWAIT`|非阻塞模式，无数据时立即返回 `EAGAIN`|
|`MSG_WAITALL`|等待完整 `len` 字节数据（除非出错或对方关闭连接）|
|`MSG_PEEK`|查看数据而不从缓冲区移除|

# `shutdown`函数
`shutdown()` 用于**关闭套接字的部分或全部数据传输通道**，它比 `close()` 提供了更精细的控制，允许你独立关闭读、写或同时关闭。
```c
#include <sys/socket.h>
int shutdown(int sockfd, int how);
```
**返回值**:  成功返回 **0**,  失败返回 **-1**，并设置 `errno` 错误码

## `how` — 关闭方式

| 常量            | 值   | 说明                |
| ------------- | --- | ----------------- |
| **SHUT_RD**   | 0   | 关闭读通道：不能再从套接字接收数据 |
| **SHUT_WR**   | 1   | 关闭写通道：不能再向套接字发送数据 |
| **SHUT_RDWR** | 2   | 关闭读写通道：读和写都关闭     |

## 与四次挥手的对应关系
- `shutdown`关闭写就对应了客户端的两次握手
- `shutdown`关闭读就对应了客户端的两次握手
- 因此两次`shutdown`就对应了四次挥手
