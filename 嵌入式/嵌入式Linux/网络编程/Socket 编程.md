# 基本概念
## `Socket`的组成
- 网络地址：通常是 IP 地址，用于标识网络上的设备。
- 端口号：用于标识设备上的特定应用或进程。端口号是一个 `16` 位的数字，范围从 `0` 到 `65535`。
- 协议：如 `TCP`（传输控制协议）或 `UDP`（用户数据报协议），定义了数据传输的规则和格式。

## `Socket`类型
- **流套接字（`Stream Sockets`）**：基于 `TCP` 协议，提供面向连接、可靠的数据传输服务。数据像流水一样连续传输，接收方按发送顺序接收数据，适用于需要准确无误传输数据的应用，如网页服务器。
- **数据报套接字（`Datagram Sockets`）**：基于 `UDP` 协议，提供无连接的数据传输服务。每个报文段独立传输，可能会丢失或无法保证顺序，适用于对传输速度要求高但可以容忍一定丢包率的应用，如在线视频会议。


# 本地字节序和网络字节序
字节序（`Endianness`）是指**多字节数据在内存中的存储顺序**。网络通信要求所有数据使用统一的字节序，否则不同架构的计算机之间无法正确解析数据。
## 什么是字节序？
一个多字节整数（如 `0x12345678`）在内存中有两种存储方式：

| 字节序                    | 存储方式       | 示例（`0x12345678`）                              |
| ---------------------- | ---------- | --------------------------------------------- |
| **大端序（Big-Endian）**    | 高位字节存储在低地址 | 地址 0: `0x12`, 1: `0x34`, 2: `0x56`, 3: `0x78` |
| **小端序（Little-Endian）** | 低位字节存储在低地址 | 地址 0: `0x78`, 1: `0x56`, 2: `0x34`, 3: `0x12` |
## 为什么需要字节序转换
| 系统      | 字节序 | 存储 `0x12345678` |
| ------- | --- | --------------- |
| x86     | 小端序 | `78 56 34 12`   |
| ARM     | 小端序 | `78 56 34 12`   |
| 网络传输    | 大端序 | `12 34 56 78`   |
| 某些嵌入式平台 | 大端序 | `12 34 56 78`   |
**如果直接发送内存数据**：
- 小端序机器发送 `0x12345678`，网络上是 `78 56 34 12`。
- 大端序机器收到后，按大端序解析为 `0x78563412`。
- 数据完全错乱！
**解决方案**：发送前统一转换为**网络字节序（大端序）**，接收后再转换为**本地字节序**。

## 端口字节序转换函数
| 函数        | 作用                               | 数据长度 |
| --------- | -------------------------------- | ---- |
| `htons()` | 主机序 → 网络序（Host to Network Short） | 16 位 |
| `htonl()` | 主机序 → 网络序（Host to Network Long）  | 32 位 |
| `ntohs()` | 网络序 → 主机序（Network to Host Short） | 16 位 |
| `ntohl()` | 网络序 → 主机序（Network to Host Long）  | 32 位 |

## `IP`地址转字节序转换
| 函数                | 方向               | 输入                    | 输出                 | 返回值                            | 支持协议        |
| ----------------- | ---------------- | --------------------- | ------------------ | ------------------------------ | ----------- |
| **`inet_pton()`** | 字符串 → 二进制（网络字节序） | `const char *`（点分十进制） | `struct in_addr *` | `1`（成功）、`0`（格式无效）、`-1`（地址族不支持） | IPv4 / IPv6 |
| **`inet_ntop()`** | 二进制（网络字节序）→ 字符串  | `struct in_addr *`    | `char *`（点分十进制）    | 成功返回 `dst` 指针，失败返回 `NULL`      | IPv4 / IPv6 |



# 公共开发函数
## `Socket` 函数
```c
#include <sys/socket.h>
int socket(int domain, int type, int protocol);
```
**返回值**：
- 成功：返回一个**非负整数**（套接字文件描述符）。
- 失败：返回 **-1**，并设置 `errno` 错误码。

### `domain`协议族
| 常量                         | 说明                                     |
| -------------------------- | -------------------------------------- |
| **AF_INET**                | `IPv4` 地址族（最常用）                        |
| **AF_INET6**               | `IPv6` 地址族                             |
| **AF_UNIX** / **AF_LOCAL** | 本地通信（`Unix Domain Socket`），用于同一主机进程间通信 |
| **AF_PACKET**              | 链路层通信（直接操作数据包）                         |
| **AF_NETLINK**             | 内核与用户空间通信（如 `ip` 命令）                   |
- **几乎总是使用`AF_INET`或者`AF_INET6`**

### `type`套接字类型
| 常量                 | 说明                                |
| ------------------ | --------------------------------- |
| **SOCK_STREAM**    | 流式套接字，基于 `TCP`，面向连接、可靠传输          |
| **SOCK_DGRAM**     | 数据报套接字，基于 `UDP`，无连接、不可靠传输         |
| **SOCK_RAW**       | 原始套接字，允许直接访问 `IP` 层（如构造 `ICMP` 包） |
| **SOCK_SEQPACKET** | 顺序包套接字，面向连接，但保留消息边界               |
| **SOCK_RDM**       | 可靠数据报，面向消息的可靠传输（较少使用）             |
- **几乎总是使用`SOCK_STREAM`或者`SOCK_DGRAM`**

### `protocol`协议
| 协议常量             | 说明       | 对应组合                                                            |
| ---------------- | -------- | --------------------------------------------------------------- |
| `0`              | 自动选择默认协议 | `AF_INET + SOCK_STREAM` → TCP  <br>`AF_INET + SOCK_DGRAM` → UDP |
| **IPPROTO_TCP**  | 显式指定 TCP | `AF_INET + SOCK_STREAM`                                         |
| **IPPROTO_UDP**  | 显式指定 UDP | `AF_INET + SOCK_DGRAM`                                          |
| **IPPROTO_ICMP** | ICMP 协议  | 通常配合 `SOCK_RAW` 使用                                              |
| **IPPROTO_RAW**  | 原始 IP 包  | 配合 `SOCK_RAW`                                                   |
|                  |          |                                                                 |
- **几乎总是设 `0`，让系统自动选择**


## `bind`函数
```c
#include <sys/socket.h>
int bind(int sockfd, const struct sockaddr *addr, socklen_t addrlen);
```
**返回值**：
- 成功：返回 **0**
- 失败：返回 **-1**，并设置 `errno` 错误码

### 参数详解
- `sockfd` 套接字文件描述符, 由 `socket()` 函数返回的套接字描述符

- `addr`地址结构体指针
指定要绑定的 IP 地址和端口号
```c
struct sockaddr_in {
    sa_family_t    sin_family;   // 地址族（AF_INET）
    in_port_t      sin_port;     // 端口号（网络字节序）
    struct in_addr sin_addr;     // IP 地址
};
```
**`struct sockaddr` 是一个通用结构**，实际使用时需要将 `struct sockaddr_in` 强制转换为 `struct sockaddr`

- `addrlen`:  `sizeof(struct sockaddr_in)`
