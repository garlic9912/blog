竞态条件，就是两个或两个以上的线程，**同时访问同一块共享内存**（全局变量、堆数据、外设寄存器等），**并且至少有一个线程在修改它**（写操作），导致最终结果依赖于线程执行的“时间顺序”。
简单说：谁先跑、谁后跑，决定了结果是正确还是错误。

# 互斥锁
| 操作                | 函数（`POSIX`线程）             | 描述                         |
| ----------------- | ------------------------- | -------------------------- |
| **加锁（Lock）**      | `pthread_mutex_lock()`    | 尝试获取锁。如果锁已被占用，线程阻塞等待。      |
| **尝试加锁（Trylock）** | `pthread_mutex_trylock()` | 尝试获取锁。如果锁已被占用，立即返回错误（不阻塞）。 |
| **解锁（Unlock）**    | `pthread_mutex_unlock()`  | 释放锁，让其他等待的线程可以获取。          |
## 初始化
- 静态初始化(编译时初始化)
```c
#include <pthread.h>

pthread_mutex_t mutex = PTHREAD_MUTEX_INITIALIZER;
```

- 动态初始化(运行时初始化)
```c
pthread_mutex_t mutex;
pthread_mutex_init(&mutex, NULL);  // NULL = 默认属性
```

## 加锁 & 解锁
```c
// 加锁（阻塞）
pthread_mutex_lock(&mutex);

// 临界区：在这里操作共享资源
count++;

// 解锁
pthread_mutex_unlock(&mutex);
```

## 完整示例
```c
#include <stdio.h>
#include <pthread.h>

int counter = 0;
pthread_mutex_t mutex = PTHREAD_MUTEX_INITIALIZER;

void* thread_func(void* arg) {
    for (int i = 0; i < 10000; i++) {
        pthread_mutex_lock(&mutex);    // 加锁
        counter++;                     // 临界区
        pthread_mutex_unlock(&mutex);  // 解锁
    }
    return NULL;
}

int main() {
    pthread_t t1, t2;
    pthread_create(&t1, NULL, thread_func, NULL);
    pthread_create(&t2, NULL, thread_func, NULL);
    
    pthread_join(t1, NULL);
    pthread_join(t2, NULL);
    
    printf("counter = %d\n", counter);  // 正确输出 20000
    return 0;
}
```

# 读写锁
读写锁是互斥锁的升级版，它区分了“读操作”和“写操作”，允许**多个读者同时访问**，但**写者独占访问**。这种设计在“读多写少”的场景中能大幅提升并发性能。
## 初始化读写锁
```c
#include <pthread.h>

// 静态初始化
pthread_rwlock_t rwlock = PTHREAD_RWLOCK_INITIALIZER;

// 动态初始化
pthread_rwlock_t rwlock;
pthread_rwlock_init(&rwlock, NULL);   // NULL = 默认属性
```
- 分为静态初始化和动态初始化

## 加锁 & 解锁
### 加读锁
```c
int pthread_rwlock_rdlock(pthread_rwlock_t *rwlock);
int pthread_rwlock_tryrdlock(pthread_rwlock_t *rwlock);
```
- `rdlock`：阻塞直到获取读锁。
- `tryrdlock`：尝试获取，如果已被写锁占用则立即返回 `EBUSY`

### 加写锁
```c
int pthread_rwlock_wrlock(pthread_rwlock_t *rwlock);
int pthread_rwlock_trywrlock(pthread_rwlock_t *rwlock);
```
- `wrlock`：阻塞直到获取写锁。
- `trywrlock`：尝试获取，如果已被其他锁占用则立即返回 `EBUSY`

### 解锁
```c
int pthread_rwlock_unlock(pthread_rwlock_t *rwlock);
```
无论当前是读锁还是写锁，都用同一个函数解锁。

### 销毁锁
```c
pthread_rwlock_destroy(&rwlock);
```


# 自旋锁
自旋锁是嵌入式开发中极其重要的一种同步机制。它的核心思想非常直接：当一个线程试图获取一个已经被其他线程持有的锁时，**它不会去阻塞**，而是会“在原地打转”，**不停地循环检查锁是否被释放**

## 使用方法
```c
#include <pthread.h>

// 创建
pthread_spinlock_t spinlock;

// 初始化
pthread_spin_init(&spinlock, PTHREAD_PROCESS_PRIVATE);

// 加锁
pthread_spin_lock(&spinlock);
// 解锁
pthread_spin_unlock(&spinlock);

// 销毁
pthread_spin_destroy(&spinlock);
```

## 注意事项
在**单核处理器**上，如果持有自旋锁的线程被抢占（比如时间片用完），而其他线程又在循环等待同一个锁，就会导致**死锁**
**在单核上，使用自旋锁时必须配合以下措施：**
1. **关中断**：防止中断处理程序争抢锁，导致死锁。
2. **禁用抢占**：确保持锁线程不会被其他线程抢占（在`Linux`中通过 `preempt_disable()`）。



# 条件变量
**条件变量是线程同步中用于“等待某个条件成立”的机制。** 它总是与互斥锁配合使用，让线程在条件不满足时进入睡眠，避免忙等待（Busy Waiting），从而高效地利用CPU资源。

## 核心组件
| 组件                           | 作用               |
| ---------------------------- | ---------------- |
| **互斥锁（Mutex）**               | 保护共享数据的访问，防止竞态条件 |
| **条件变量（Condition Variable）** | 负责线程的等待与唤醒       |

## 核心操作
```c
#include <pthread.h>

// 1. 初始化条件变量
pthread_cond_t cond = PTHREAD_COND_INITIALIZER;  // 静态
// 或
pthread_cond_init(&cond, NULL);  // 动态

// 2. 等待条件（一直阻塞, 等待被唤醒）
pthread_cond_wait(&cond, &mutex);

// 3. 等待条件（带超时, 超时则不再等待并返回）
pthread_cond_timedwait(&cond, &mutex, &abstime);

// 4. 唤醒一个等待线程
pthread_cond_signal(&cond);

// 5. 唤醒所有等待线程
pthread_cond_broadcast(&cond);

// 6. 销毁条件变量
pthread_cond_destroy(&cond);
```

## 代码示例
```c
#include <pthread.h>

pthread_mutex_t mutex;
pthread_cond_t cond;
int ready = 0;

void thread1(void *param) {
    pthread_mutex_lock(&mutex);
    while (!ready) {
	    // 条件不成立, 暂停线程
        pthread_cond_wait(&cond, &mutex);
    }
    // 条件成立，执行操作
    pthread_mutex_unlock(&mutex);
}

void thread2(void *param) {
    pthread_mutex_lock(&mutex);
    ready = 1;
    
    // 唤醒一个暂停的线程
    pthread_cond_signal(&cond);
    pthread_mutex_unlock(&mutex);
}
```
- 条件变量相当于一个"暂停建", 如果条件不满足则暂停当前线程, 等待其他线程处理完在回来执行
