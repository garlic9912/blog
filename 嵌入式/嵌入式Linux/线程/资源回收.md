# 阻塞: `pthread_join()`
```c
int pthread_join(pthread_t thread, void **retval);
```
- 阻塞等待指定线程结束
- `retval` 用于接收线程入口函数的返回值（或 `pthread_exit` 的参数）

## ==注意事项==
- **必须对每个可连接的线程调用 `pthread_join`**，否则线程结束后资源无法回收，造成**内存泄漏**。
- `pthread_join` 是阻塞的，如果线程尚未结束，调用线程会一直等待。
- 如果不需要线程返回值，`retval` 可以传 `NULL`。


# 分离: `pthread_detach()`
`pthread_detach` 是线程生命周期管理中最常用的工具之一。它告诉系统：这个线程结束时会自动回收资源，**但我不需要等待它**。这相当于把线程的清理工作“外包”给了操作系统。

|状态|行为|回收方式|
|---|---|---|
|**可连接（Joinable，默认）**|线程结束后资源不释放，等待 `pthread_join`|必须调用 `pthread_join`|
|**分离（Detached）**|线程结束后立即自动释放资源|无需 `pthread_join`，自动回收|

## 函数原型
```c
#include <pthread.h>

int pthread_detach(pthread_t thread);
```
- **作用**：将指定的线程标记为分离状态
- **参数**：`thread` 是要分离的线程 ID
- **返回值**：成功返回 `0`；失败返回错误码（如 `ESRCH` 表示线程不存在，`EINVAL` 表示线程已经是分离状态）

## 代码示例
```c
#include <stdio.h>
#include <pthread.h>
#include <unistd.h>

void *thread_func(void *arg) {
    printf("子线程开始执行\n");
    sleep(2);
    printf("子线程结束（资源自动回收）\n");
    return NULL;
}

int main() {
    pthread_t tid;
    
    // 1. 创建线程（默认是可连接的）
    pthread_create(&tid, NULL, thread_func, NULL);
    
    // 2. 分离线程
    pthread_detach(tid);
    
    // 3. 无需 pthread_join
    printf("主线程继续执行\n");
    sleep(3);
    printf("主线程退出\n");
    
    return 0;
}
```
