线程是程序执行的最小单元，一个进程可以包含多个线程，它们共享进程的地址空间、文件描述符等资源。在嵌入式 Linux 开发中，多线程是实现并发处理的核心手段。

# 线程创建
```c
#include <pthread.h>

int pthread_create(pthread_t *thread, 
                   const pthread_attr_t *attr,
                   void *(*start_routine)(void *),
                   void *arg);
```

|参数|说明|
|---|---|
|`thread`|指向 `pthread_t` 的指针，用于存储线程 ID|
|`attr`|线程属性（通常为 `NULL` 表示默认属性）|
|`start_routine`|线程入口函数，原型为 `void *(void *)`|
|`arg`|传递给入口函数的参数|
**返回值**：
- `0`：成功
- 非 `0`：错误码（如 `EAGAIN` 资源不足）

## 代码示例
```c
#include <stdio.h>
#include <pthread.h>
#include <unistd.h>

// 线程入口函数
void *thread_func(void *arg) {
    int *num = (int *)arg;
    printf("线程: 接收到参数 %d\n", *num);
    return NULL;
}

int main() {
    pthread_t tid;
    int value = 123;
    
    // 创建线程
    if (pthread_create(&tid, NULL, thread_func, &value) != 0) {
        perror("pthread_create");
        return 1;
    }
    
    // 等待线程结束
    pthread_join(tid, NULL);
    printf("主线程: 线程已结束\n");
    
    return 0;
}
```
## ==注意事项==
- **`pthread_create` 只负责创建线程**，不保证线程立即执行（取决于调度）.    
- **线程入口函数只能有唯一的参数 `void *arg`**: 如果要传递多个参数, 需要借助结构体来完成
- **参数 `arg` 的生命周期**：必须保证在线程使用期间有效（不要传递栈上的临时变量地址，除非在 `pthread_join` 之前不会被销毁）。


# 线程终止
## `return`终止
```c
void *thread_func(void *arg) {
    // 执行任务
    return NULL;  // 返回后线程终止
}
```

## 显示调用`pthread_exit()`
```c
void *thread_func(void *arg) {
    // 执行部分任务
    pthread_exit(NULL);  // 主动终止
    // 这行不会执行
}
```

## 被其他线程取消
```c
// 主线程中
pthread_cancel(tid);  // 请求取消 tid 线程
```
