# 向量

**向量**是有序排列的数字列表，通常表示为一个行或列的数组。例如：

```textile
v = [0.25, 0.80, -0.13, 0.97, 0.42]
```

数据库存储这些向量后，可以通过**相似度计算**（余弦相似度、欧氏距离等）找到与查询向量最接近的向量，从而返回语义上最相似的结果。

想象一个**物品特征表**，每个物品用 3 个特征描述：甜度、脆度、颜色深度。那么：

- 苹果 = [7, 8, 5]

- 梨 = [6, 7, 4]

- 桌子 = [0, 0, 10] （完全不甜不脆，但颜色深）

在这个三维空间中，苹果和梨离得近（相似），苹果和桌子离得远（不相似）。**每个物品的特征列表就是一个向量**。现实中的向量维度可能高达 1024、1536 甚至更高，这些维度无法用直观的名称定义，由神经网络自动学习。

# 相似度

## 1.余弦相似度

余弦相似度通过计算两个向量在向量空间中的**夹角余弦值**来衡量它们之间的相似程度，其关注的是向量的**方向差异**而非长度差异。

| 取值范围 | 含义      |
| ---- | ------- |
| 1    | 方向完全相同  |
| 0    | 正交，方向无关 |
| -1   | 方向完全相反  |

## 2.欧式距离

欧氏距离是向量空间中两点之间的**直线距离**，它度量的是两个向量在绝对位置上的差异程度。

| 取值范围 | 含义   |
| ---- | ---- |
| 0    | 完全相同 |
| 越大   | 差异越大 |

## 3.点积

点积是两个向量对应分量乘积之和，它同时考虑了向量的**方向和模长**信息，是最基本的向量运算之一。

| 取值范围 | 含义              |
| ---- | --------------- |
| 正值   | 方向夹角小于 90°，总体相近 |
| 0    | 方向正交（垂直）        |
| 负值   | 方向夹角大于 90°，总体相异 |

## 使用场景

三种计算方法有不同, 使用的场景也不同, 以下是经典的使用场景

| 推荐方法      | 经典使用场景           | 核心原因              |
| --------- | ---------------- | ----------------- |
| **余弦相似度** | 文本相似度（文档匹配、句子语义） | 忽略文档长度，只关注语义方向    |
| **欧氏距离**  | 图像检索、人脸识别        | 像素/特征值的绝对差异具有物理意义 |
| **点积**    | 神经网络（注意力机制、全连接层） | 计算速度最快，硬件友好       |
