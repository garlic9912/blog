# ANN

ANN（Approximate Nearest Neighbor）是一类**牺牲一定精度来换取检索速度**的算法，用于在高维空间中快速找到查询点的近似最近邻。

## 传统方法的缺点

**传统方法的困境**：

- **精确搜索（暴力扫描）**：计算查询点与所有点的欧式距离(余弦相似度, 点积)，复杂度 O(N), 当数据量达到百万、亿级时，计算量不可接受

**维度灾难的影响**：

- 高维空间中，树形索引（KD-Tree、B-Tree）退化为线性扫描

- 传统索引结构在高维场景下失效

## ANN 算法家族

| 算法类型     | 代表算法            | 核心思想      | 特点         |
| -------- | --------------- | --------- | ---------- |
| 基于树的     | KD-Tree、RP-Tree | 空间划分      | 高维退化严重     |
| 基于哈希的    | LSH（局部敏感哈希）     | 哈希函数保持相似性 | 内存效率高，精度较低 |
| 基于量化的    | PQ（乘积量化）        | 向量压缩编码    | 内存占用极小     |
| **基于图的** | **HNSW**        | 构建近邻图导航   | 精度和速度平衡最好  |

## ANN 的优势

| 维度     | 精确搜索       | ANN                 |
| ------ | ---------- | ------------------- |
| 召回率    | 100%       | 95%-99%（可能漏掉真正的最近邻） |
| 查询速度   | 慢（O(N)）    | 快（O(log N) 或 O(1)）  |
| 索引构建时间 | 无或很长       | 中等                  |
| 内存占用   | 高（需存储所有向量） | 高（加上索引结构）           |

# HNSW

HNSW（Hierarchical Navigable Small World）是目前工业界最流行、性能最优的 ANN 算法之一，它通过构建**多层图结构**实现对数级别的高效检索。

## 核心思想

HNSW的核心创新是构建**多层图结构**，灵感来源于跳表（Skip List）

```textile
// 注意每一层是一个图, 这里仅展示侧面
Layer 2 (顶层):    A ←→ B          (仅2个节点)
                    ↓   ↙
Layer 1 (中层):    A ←→ B ←→ C ←→ D   (4个节点)
                    ↓   ↓   ↓   ↓
Layer 0 (底层):    A ←→ B ←→ C ←→ D ←→ E ←→ F ←→ G  (所有节点)
```

## 单图层查找算法

给 HNSW 算法一个查询向量，从顶层随机进入一个节点，然后在每层贪心地查找，最终到达底层，返回结果。**贪心**在 HNSW 中指的是：在每一层图中，每一步都移动到当前节点的邻居中距离查询向量最近的那个节点，直到没有更近的邻居为止。

| 维度       | 说明                       |
| -------- | ------------------------ |
| **决策依据** | 只看当前节点的直接邻居，不展望更远的节点     |
| **移动方向** | 始终向**距离查询点更近**的方向移动，绝不后退 |
| **停止条件** | 所有邻居都比当前节点离查询点更远（局部最优）   |
| **结果性质** | **近似**最近邻，不保证全局最优        |

HNSW 的贪心搜索与 Dijkstra 等算法中的“贪心策略”本质相同：**只做当下最优选择，不为未来做权衡**。区别在于 HNSW 不需要路径累积成本，只比较单步的向量距离。
