# Docker

## Docker 的作用

- **定义**：开源的应用容器引擎

- **核心价值**：标准化（应用打包、交付、运行的全流程标准化）

## 核心概念

| 概念                | 性质        | 说明                                         |
| ----------------- | --------- | ------------------------------------------ |
| **Dockerfile**    | 构建配方（声明式） | 描述如何从基础镜像逐步构建目标镜像，包含基础镜像、依赖安装、代码拷贝、启动命令等指令 |
| **Image（镜像）**     | 静态模板/安装包  | 只读的、分层构建的文件系统快照，可存储、分享、版本化管理               |
| **Container（容器）** | 动态运行实例    | 镜像的一个可写层实例，具有独立的进程、网络、文件系统视图，可启动、停止、删除     |

## 核心功能

**打包（Containerization）**

- 将代码 + 依赖库 + 配置文件 + 环境变量打包为统一单位（镜像）

- 类比：物理世界的集装箱

**隔离（Isolation）**

- 利用 Linux 内核的 Namespace 和 Cgroups 技术

- 同一宿主机上可运行多个相互隔离的容器

- 解决不同应用对运行时环境的版本冲突问题（如 Python 2.7 与 3.8）

**可移植运行（Portable Execution）**

- 镜像在任何安装 Docker 引擎的机器上均可秒级启动

## 解决的痛点

| 痛点       | Docker 解决方案                                                |
| -------- | ---------------------------------------------------------- |
| 环境不一致    | 环境+代码一起打包，镜像确保运行环境完全一致                                     |
| 虚拟机资源开销大 | 容器共享宿主机内核，无需完整 Guest OS，占用内存（MB vs GB）、启动时间（秒 vs 分钟）均优于虚拟机 |
| 微服务部署复杂  | 每个微服务独立容器，独立部署、扩展、升级                                       |
| 自动化运维    | 作为流水线的标准化构件，从代码提交到自动构建、测试、部署形成闭环s                          |
