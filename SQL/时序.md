# TIMESTAMP 字面量
`TIMESTAMP` 是 SQL 中表示日期和时间的数据类型，精确到秒或小数秒（如毫秒、微秒）。`TIMESTAMP` 字面量用于在 SQL 语句中直接书写一个时间点常量。
```sql
TIMESTAMP 'YYYY-MM-DD HH:MI:SS'
TIMESTAMP 'YYYY-MM-DD HH:MI:SS.FFF'
```
- ==必须使用单引号包裹日期时间字符串==。
- 日期部分与时间部分之间有一个空格。
- 字符串格式必须严格遵循 ISO 格式（`YYYY-MM-DD HH:MI:SS`），否则数据库可能报错或产生隐式转换。
- 不要省略 `TIMESTAMP` 关键字，否则字符串可能被当作普通字符串，引发隐式转换问题或索引失效。

## 示例
```sql
-- 基本用法
SELECT TIMESTAMP '2026-06-15 08:00:00';

-- 带毫秒
SELECT TIMESTAMP '2026-06-15 08:00:00.123';

-- 在 WHERE 子句中使用
SELECT * FROM readings
WHERE ts > TIMESTAMP '2026-06-15 00:00:00';
```


# INTERVAL 数据类型
`INTERVAL` 表示一段时间长度（时间间隔），用于日期时间的加减运算。它可以是常量，也可以是动态计算的表达式。
```sql
INTERVAL '数值' 单位          -- 常量形式
INTERVAL (表达式) 单位        -- 动态形式（部分数据库支持括号）
```
**常用单位**：`YEAR`、`MONTH`、`DAY`、`HOUR`、`MINUTE`、`SECOND`，以及复合单位如 `HOUR TO MINUTE`

## 示例
- **常量 INTERVAL**
```sql
SELECT INTERVAL '5' HOUR;            -- 5 小时
SELECT INTERVAL '2' DAY;              -- 2 天
SELECT INTERVAL '1:30' HOUR TO MINUTE; -- 1 小时 30 分
```
- **动态 INTERVAL（列值参与计算）**
```sql
-- up.acc_time 是数值列，假设值为 3
SELECT INTERVAL (up.acc_time + 2) HOUR FROM up;
-- 结果为 INTERVAL '5' HOUR
```

# 时序计算
`TIMESTAMP` 可以与 `INTERVAL` 进行加（`+`）或减（`-`）运算，结果仍为 `TIMESTAMP`。这用于在某个时间点基础上向前或向后推移一段时间。
## 运算规则
- `timestamp + interval` → 未来时间点
- `timestamp - interval` → 过去时间点
- `timestamp - timestamp` → 得到 `INTERVAL`（两个时间点的差）

## 示例
```sql
-- 加法：向前推 3 小时
SELECT TIMESTAMP '2026-06-15 08:00:00' + INTERVAL '3' HOUR;
-- 结果：2026-06-15 11:00:00

-- 减法：向后推 2 天
SELECT TIMESTAMP '2026-06-15 08:00:00' - INTERVAL '2' DAY;
-- 结果：2026-06-13 08:00:00

-- 动态间隔：减去 (up.acc_time + 2) 小时
SELECT TIMESTAMP '2026-06-15 08:00:00' - INTERVAL (up.acc_time + 2) HOUR
FROM up;
```
