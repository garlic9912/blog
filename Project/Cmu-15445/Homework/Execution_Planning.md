# 元组嵌套循环和块嵌套循环的区别

两者最大的区别在于**外层的循环粒度不同**，一个是以“一行”为单位，另一个是以“一块”为单位

- 元组嵌套循环
  
  **外层的循环单位是一个元组**
  
  - 算法逻辑：外层表**每读一元组**，就要把内层表**完整扫描一遍**。
  
  - I/O 代价：`外表总行数 × 内表页数 + 外表页数`。
  
  - 这是最暴力的方式，内层表数据在内存里反复搬运，代价最高。

- 块嵌套循环
  
  **外层的循环单位是“块”（一个内存能容纳的页面集合）。**
  
  - 算法逻辑：利用内存缓冲区，一次尽可能多地把外层的页读进来，凑成一“块”。然后对这个块里的所有行，再去匹配内层表里的一个页。
  
  - I/O 代价：`⌈外表页数 ÷ (B-2)⌉ × 内表页数 + 外表页数`。
  
  - 改进点：在这个块被处理的期间，内层页不需要反复从磁盘上读。相当于把外层的行攒起来，批量处理，分摊了读取内层的开销。

# 排序归并连接

**先排序，后归并。** 将两个表按连接键排序，然后用双指针按归并算法连接两个表

```textile
X（已排序）: [a=1, a=1, a=2, a=3, ...]
Z（已排序）: [a=1, a=2, a=2, a=3, ...]

两个指针同步移动，遇到相同的 a 值就输出匹配组合
```

## 两个阶段

| 阶段          | 做什么                     | 代价                  |
| ----------- | ----------------------- | ------------------- |
| **第一阶段：排序** | 把两个表都按连接键排好序（通常用外部归并排序） | 如果未排序，代价很高          |
| **第二阶段：合并** | 双指针同步扫描，匹配连接            | 已排序时为 `外表页数 + 内表页数` |

## 指针移动策略

R 是外表, S 是内表

| 阶段            | 进入条件                       | 指针移动与关键操作                                                                                                                                             | 目的 / 备注                                 |
| ------------- | -------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------- | --------------------------------------- |
| **推进阶段**      | 初始状态，或上一阶段结束后 `R.a ≠ S.a`  | 比较 `R[ptr_R].a` 与 `S[ptr_S].a`                               若 `R.a < S.a` → `ptr_R++`     若 `R.a > S.a` → `ptr_S++`     若 `R.a == S.a` → 进入匹配阶段      | 使两表指针指向的键值尽量相等，不回溯                      |
| **匹配处理与标记阶段** | `R.a == S.a`               | 1. `Mark_S = ptr_S`（记录S重复组起点）                                     2. 循环：输出 `(R[ptr_R], S[ptr_S])`，然后 `ptr_S++`     3. 直到 `S[ptr_S].a ≠ R[ptr_R].a` 为止 | 将当前R元组与S中所有相同键值元组连接，并移动S指针到下一个不同键值的位置   |
| **外表推进与回溯阶段** | 匹配阶段结束，`ptr_R++` 移动到下一个R元组 | **情况A**（新 `R.a` == 旧值）：`ptr_S = Mark_S`（回溯到S重复组开头）然后重新进入匹配阶段  **情况B**（新 `R.a` ≠ 旧值）： `ptr_S` 保持原位，然后返回推进阶段                                            | 处理R中重复键值：相同R值需重复匹配同一组S；不同R值则S指针不回溯，继续推进 |

## 适用场景

| 情况                    | 是否适用           |
| --------------------- | -------------- |
| 两个表已按连接键排序（如有聚集索引）    | ✅ 最优           |
| 等值连接                  | ✅              |
| 非等值连接（`<`, `>`, `!=`） | ❌ 不适用          |
| 一个表很小                 | Hash Join 可能更优 |



# 连接操作中内外表选择的性能影响分析

在数据库查询优化中，连接两张表时，哪张表作为“外表”以及哪张表作为“内表”，对性能的影响取决于算法类型、内存容量以及索引情况

## 核心概念

* 外表 (Outer Table)：扫描一次，对于其中的每一行/块，去寻找匹配的内表行。
* 内表 (Inner Table)：可能被扫描多次（取决于缓冲区大小和算法）

## 内存充足的情况

结论：I/O 成本无显著差异，性能几乎等同

I/O 分析：因为缓冲区足够大，两张表都只需从磁盘读取到内存一次. 总 I/O 成本 = 表 A 页数 + 表 B 页数。

## 内存不足, 必须借助磁盘 IO

| 优先级   | 判断依据                          | 行动                           |
| ----- | ----------------------------- | ---------------------------- |
| **1** | 哪个表 **能完全装入内存** (`表页数 ≤ K`)   | 该表做 **外表**（块嵌套循环时, 内表只用扫描一次） |
| **2** | 若都装不下，比较 **分块数** `⌈表/K⌉` 谁更小？ | **分块数更小的表** 做外表              |
| **3** | 若分块数相同，比较 **页面数** 谁更大         | **页面数更大的表** 做外表              |



# 区域映射

**区域映射**（Zone Map，也叫 Min-Max 索引或块级索引）是一种数据库存储层的**粗粒度索引技术**。它的核心思想是：将数据按物理块（比如磁盘上的页、数据块）分组，为每个块预先记录该块中某列（或多列）的**最小值（min）** 和**最大值（max）** 等统计信息。

## 工作原理

1. 查询执行时，数据库根据过滤条件（例如 `WHERE age > 30`）检查每个块的区域映射。

2. 如果查询条件与某个块的最小/最大值范围**无交集**（例如块中 age 的最大值 ≤ 30），则该块被**直接跳过**，无需读取其中的实际数据。

3. 只有那些范围可能包含匹配行的块才会被访问。

## 典型作用

- **快速过滤**：对大表扫描时，大幅减少需要读取的数据块数量。

- **聚合优化**：若需要 `MIN` 或 `MAX`，可从区域映射中直接获取该区域的最大值最小值, 然后直接返回，无需读取数据。
